SECTION 1: Executive Overview

This document provides the section-wise deployment documentation for the Monthly Payables Management & Tracking System. Built with a high-density, dark financial dashboard theme, the application is tailored for accountants and business controllers monitoring outgoing vendor schedules and monthly settlements.

SECTION 2: System Architecture Stack

The application is bootstrapped inside the Next.js App Router workspace, utilising typescript interfaces and modern React 19 primitives:

Interface & Styling

Tailwind CSS v4 engine loaded with CSS theme variables, custom dark scrollbars, and JetBrains Mono fonts for numbers rendering.

Data Operations Layer

A smart adapter routing DB queries. Fallbacks to localStorage mock DB when Supabase URL environment tags are placeholder strings.

SECTION 3: Database Schema Definitions

The structured SQL files are seeded inside the repository. Tables track Core Payables, Categories, PDCs, and Loan schedules:

1. Core Payables Schema

CREATE TABLE payables (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  title TEXT NOT NULL,
  category_id uuid REFERENCES categories(id),
  vendor_name TEXT,
  amount NUMERIC(12, 3) NOT NULL,
  currency TEXT DEFAULT 'OMR',
  due_date DATE NOT NULL,
  status TEXT DEFAULT 'pending' CHECK (status IN ('pending', 'paid', 'overdue', 'cancelled')),
  recurrence TEXT DEFAULT 'once' CHECK (recurrence IN ('once', 'monthly', 'quarterly', 'annual')),
  month_year TEXT NOT NULL
);

2. Post-Dated Cheques (PDC) Schema

CREATE TABLE pdcs (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  payable_id uuid REFERENCES payables(id) ON DELETE CASCADE,
  cheque_no TEXT NOT NULL,
  bank_name TEXT,
  cheque_date DATE NOT NULL,
  status TEXT DEFAULT 'pending' CHECK (status IN ('pending', 'presented', 'cleared', 'bounced'))
);
